# 第八章：误导线索库（Misdirection Library）

## 8.1 设计目标

误导线索不是为了欺骗玩家。

误导线索的目标是制造悬念、冲突、反差和推理张力。

误导必须遵守一个原则：

```text
误导可以改变玩家的判断方向，
但不能破坏案件真相。
```

也就是说，误导线索必须能被后续关键线索纠正。

禁止生成无法解释的误导。

---

## 8.2 误导线索的核心作用

误导线索承担以下功能：

- 制造第一印象
- 放大嫌疑人冲突
- 掩盖关键事实
- 推迟真相暴露
- 诱导玩家形成错误假设
- 为反转提供前置铺垫

误导线索必须服务于真相树。

它不能独立存在。

---

## 8.3 误导线索分类

### 8.3.1 身份误导

让玩家误判某个NPC的身份。

例如：

```text
看起来是普通租客
实际上是调查记者
```

```text
看起来是受害者家属
实际上是事件参与者
```

适用场景：

- 报案人
- 目击者
- 房东
- 保安
- 同事
- 同学

---

### 8.3.2 动机误导

让玩家误判NPC为什么行动。

例如：

```text
看起来是为了钱
实际上是为了隐藏旧案
```

```text
看起来是情感纠纷
实际上是数据泄露
```

---

### 8.3.3 时间误导

让玩家误判事件发生时间。

例如：

```text
监控显示23:10有人进入房间
但真正进入时间是22:47
```

```text
最后一条消息是凌晨2:13
但手机早已不在本人手上
```

时间误导必须有校正线索。

校正线索可以是：

- 电梯记录
- 门禁记录
- GPS记录
- 外卖订单
- 通话记录
- 监控时间戳异常

---

### 8.3.4 空间误导

让玩家误判事件发生地点。

例如：

```text
看起来发生在240号单位
实际上发生在24楼走廊
```

```text
看起来在大学宿舍
实际上在校外短租公寓
```

空间误导适合搭配地图、监控、定位记录。

---

### 8.3.5 证据误导

让某个证据看起来支持错误结论。

例如：

```text
房间里有血迹
但血迹来自旧伤或动物
```

```text
聊天记录像勒索
但实际上是暗号
```

证据误导必须可被验证。

禁止生成“因为AI说是假的所以是假的”的证据。

---

### 8.3.6 情绪误导

利用NPC情绪影响玩家判断。

例如：

```text
NPC表现得很害怕
玩家误以为他是受害者
实际上他害怕的是自己的谎言被发现
```

```text
NPC非常愤怒
玩家误以为他有杀意
实际上他只是想保住工作
```

情绪误导适合聊天、电话、语音留言场景。

---

### 8.3.7 超自然误导

让案件看起来像都市传说或异常事件。

例如：

```text
不存在的房间发来短信
实际上是旧门禁系统编号错误
```

```text
午夜电梯自动停在24楼
实际上是有人远程调用维修模式
```

该类型适合F级到B级案件。

A级以上可以反向处理：

```text
看起来是人为
实际上存在异常
```

---

### 8.3.8 系统误导

让玩家误信某个数字系统。

例如：

```text
物业系统显示房间空置
但有人一直在使用临时门禁
```

```text
GPS显示目标在海湾区
但手机被丢在共享车内
```

系统误导适合高度数字化的白港市。

---

## 8.4 标准误导模板

### 模板一：看起来像凶杀，实际上是失踪

```json
{
  "misdirection_type": "evidence",
  "surface_conclusion": "目标可能已经遇害",
  "actual_truth": "目标主动失踪，现场血迹来自旧伤",
  "supporting_false_clues": ["CLUE_BLOOD_001", "CLUE_BROKEN_PHONE_001"],
  "correction_clues": ["CLUE_GPS_002", "CLUE_ELEVATOR_003"],
  "used_in_act": "第一幕",
  "resolved_in_act": "第三幕"
}
```

---

### 模板二：看起来像超自然，实际上是人为

```json
{
  "misdirection_type": "supernatural",
  "surface_conclusion": "240号单位存在异常住户",
  "actual_truth": "有人利用旧门禁编号制造幽灵房源",
  "supporting_false_clues": ["CLUE_EMPTY_ROOM_PHOTO", "CLUE_MIDNIGHT_MESSAGE"],
  "correction_clues": ["CLUE_ACCESS_LOG", "CLUE_PROPERTY_BACKUP"],
  "used_in_act": "第一幕",
  "resolved_in_act": "第二幕"
}
```

---

### 模板三：看起来像骗子，其实是真话

```json
{
  "misdirection_type": "identity",
  "surface_conclusion": "匿名爆料人是骗子",
  "actual_truth": "爆料人掌握真实关键线索，但表达方式混乱",
  "supporting_false_clues": ["CLUE_FAKE_ACCOUNT", "CLUE_INCONSISTENT_MESSAGE"],
  "correction_clues": ["CLUE_OLD_CASE_FILE", "CLUE_AUDIO_MATCH"],
  "used_in_act": "第一幕",
  "resolved_in_act": "第三幕"
}
```

---

### 模板四：看起来像受害者，其实是加害者

```json
{
  "misdirection_type": "role",
  "surface_conclusion": "报案者是受害者",
  "actual_truth": "报案者主动引导玩家调查错误方向",
  "supporting_false_clues": ["CLUE_PANIC_CALL", "CLUE_THREAT_MESSAGE"],
  "correction_clues": ["CLUE_CALL_SOURCE", "CLUE_DELETED_CHAT"],
  "used_in_act": "第一幕",
  "resolved_in_act": "第四幕"
}
```

## 8.5 误导强度等级

```text
1级：轻微误导
只影响玩家对细节的判断。

2级：方向误导
影响玩家选择调查对象或地点。

3级：核心误导
影响玩家对真相的主要判断。

4级：结构误导
影响玩家对案件类型的判断。

5级：主线误导
影响玩家对暗线、观察局或城市异常的理解。
```

F级案件建议使用1~2级误导。

D级到C级案件建议使用2~3级误导。

B级以上可以使用4级误导。

S级事件允许使用5级误导。

---

## 8.6 误导与案件等级关系

| 案件等级 | 推荐误导数量 | 推荐误导强度 |
|---|---:|---:|
| F级 | 1 | 1 |
| E级 | 1~2 | 1~2 |
| D级 | 2 | 2 |
| C级 | 2~3 | 2~3 |
| B级 | 3 | 3~4 |
| A级 | 3~4 | 4 |
| S级 | 4~5 | 4~5 |

误导数量不能过多。

10分钟体验中，过多误导会导致玩家疲劳。

---

## 8.7 误导生成字段

每条误导线索必须包含：

```json
{
  "misdirection_id": "MIS_001",
  "case_id": "CASE_001",
  "type": "time",
  "strength": 2,
  "surface_conclusion": "玩家最初会以为事件发生在凌晨2:13",
  "actual_truth": "凌晨2:13只是消息发送时间，事件实际发生在1:42",
  "related_truth_node": "TRUTH_NODE_003",
  "related_clues": ["CLUE_CHAT_001", "CLUE_TIMESTAMP_002"],
  "correction_clues": ["CLUE_ELEVATOR_004"],
  "related_npcs": ["NPC_002"],
  "introduced_scene": "SCENE_002",
  "resolved_scene": "SCENE_007",
  "player_effect": "引导玩家先调查发信人，而不是门禁记录"
}
```

---

## 8.8 误导校验规则

生成误导后，系统必须检查：

```text
1. 每条误导必须指向至少一个真相节点。
2. 每条误导必须拥有至少一个校正线索。
3. 误导不能和最终真相矛盾。
4. 误导不能依赖纯巧合。
5. 误导不能让玩家陷入无解状态。
6. 误导不能替代关键线索。
7. 误导必须能在场景中被玩家感知。
```

---

## 8.9 禁止事项

禁止：

- 为了反转而反转
- 误导线索无法解释
- NPC无理由说谎
- 时间线强行改写
- 证据前后冲突
- 关键证据只在结尾突然出现
- 玩家无法通过调查提前发现问题

允许：

- 玩家早期误判
- NPC因恐惧隐瞒
- 系统记录被篡改
- 证据有双重解释
- 同一线索前后产生不同意义

---

## 8.10 误导设计原则

误导不是假线索。

误导是半真半假的线索。

最好的误导应该满足：

```text
第一次看：它指向错误答案。

真相揭晓后再看：它其实一直在说真话。
```

这就是《暗线》的误导标准。
