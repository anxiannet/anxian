# 03 案件模板系统（Case Templates）

## 3.1 设计目标

案件模板负责快速生成短平快调查体验。

目标：

- 开局即进入冲突
- 5分钟获得关键线索
- 10分钟完成调查

---

## 3.2 标准结构

```json
{
  "case_id":"CASE_001",
  "title":"不存在的240号房",
  "hook":"24楼出现一间不存在的房间",
  "conflict":"物业系统中查不到记录",
  "truth":"门牌被人为替换",
  "twist":"目击者才是真正操作者"
}
```

---

## 3.3 基础模板

### 失踪类

- 人失踪
- 物失踪
- 记录失踪

### 身份类

- 冒名顶替
- 双重身份
- 不存在的人

### 地点类

- 不存在房间
- 消失楼层
- 异常地点

### 信息类

- 神秘短信
- 匿名举报
- 错误档案

---

## 3.4 矛盾模板

每个案件至少一个矛盾。

示例：

- 人在现场，但监控没有
- 门开过，但门禁没有记录
- 电话打出，但机主失踪
- 新闻存在，但事件不存在

---

## 3.5 反转模板

### 身份反转

嫌疑人 → 受害者

### 时间反转

先后顺序错误

### 地点反转

线索来自其他地点

### 动机反转

求助者就是策划者

---

## 3.6 线索配置

```json
{
  "scene_clues":2,
  "image_clues":2,
  "audio_clues":1,
  "npc_clues":2,
  "key_clue":1
}
```

---

## 3.7 难度等级

### 普通

1个矛盾
1个反转

### 稀有

2个矛盾
2个反转

### 隐藏

涉及都市传说

### 暗线

涉及暗线节点

---

## 3.8 短剧化规则

案件必须具备：

- 钩子
- 冲突
- 悬念
- 反差
- 反转

参考短剧节奏而非传统TRPG。

---

## 3.9 AI生成公式

```text
异常事件
+
矛盾
+
线索
+
误导
+
反转
+
真相
=
案件
```
