# 02 事件触发系统（Event Triggers）

## 2.1 设计目标

事件是《暗线》的核心驱动力。

玩家不主动找任务。

事件主动找到玩家。

目标：

- 1分钟进入事件
- 5分钟卷入谜团
- 10分钟完成一次调查

---

## 2.2 触发原则

每个事件必须满足至少一个条件：

- 存在异常
- 存在矛盾
- 存在风险
- 存在秘密
- 存在未知身份

禁止：

- 普通跑腿任务
- 纯收集任务
- 无谜团任务

---

## 2.3 事件来源

### 手机消息

- 匿名私信
- 群聊截图
- 求助信息

### 新闻

- 本地新闻
- 热门话题
- 突发事件

### NPC

- 朋友
- 邻居
- 同事
- 线人

### 暗线节点

- 神秘通知
- 异常档案
- 未知信号

---

## 2.4 触发等级

```json
{
  "trigger_level": "normal",
  "priority": 50
}
```

等级：

- normal
- rare
- hidden
- darkline

---

## 2.5 标准结构

```json
{
  "event_id":"EVT_001",
  "title":"24楼的陌生灯光",
  "trigger_type":"message",
  "summary":"有人发来一张奇怪照片。",
  "location":"白塔公寓",
  "entry_scene":"SCENE_001"
}
```

---

## 2.6 事件生命周期

触发
↓
调查
↓
发现线索
↓
推理
↓
归档
↓
后续事件

---

## 2.7 事件链

一个事件可衍生多个事件。

示例：

不存在的240号房
↓
失踪住户
↓
旧门禁记录
↓
白塔公寓档案
↓
暗线节点

---

## 2.8 AI生成规则

事件必须：

- 简单直白
- 冲突明确
- 开场有钩子
- 存在悬念
- 存在反转可能

禁止长篇背景设定。
