# 第十四章：AI生成规则（AI Generation Rules）

## 14.1 设计目标

AI生成规则用于约束大语言模型在《暗线》中的工作边界。

核心原则：

```text
系统负责真相。
AI负责表现。
```

系统生成：

- 真相树
- 隐藏真相
- 事实节点
- NPC秘密
- 关键线索
- 误导线索
- 反转节点
- 场景结构
- 玩家状态

AI生成：

- 对白
- 氛围
- 描述
- UI文案
- 图片提示词
- 语音脚本
- 新闻包装
- 档案摘要

AI不得修改系统真相。

---

## 14.2 禁止先写剧情

禁止流程：

```text
先写故事
↓
再补线索
↓
再解释真相
```

这种方式会导致：

- 线索前后矛盾
- NPC动机不稳定
- 反转突兀
- 真相无法校验
- 玩家推理无意义

必须使用以下流程：

```text
生成真相
↓
生成隐藏真相
↓
生成事实节点
↓
生成关键线索
↓
生成NPC秘密
↓
生成误导线索
↓
生成反转
↓
生成场景
↓
生成对白和表现文本
```

---

## 14.3 AI可生成内容

AI允许生成：

### 14.3.1 对白包装

例如系统提供：

```json
{
  "npc": "林知远",
  "truth": "他删除过凌晨2:13的录像",
  "emotion": "紧张",
  "lie_level": 35
}
```

AI可以生成：

```text
“我只是按流程清理旧录像。凌晨那段？我……我不记得了。”
```

AI不能生成：

```text
“录像不是我删的。”
```

除非系统允许该NPC在该场景说谎。

---

### 14.3.2 氛围描述

AI可以描述：

```text
电梯灯闪了一下，24楼的按钮没有亮，却传来一声很轻的提示音。
```

AI不能新增未注册线索：

```text
墙上突然出现血字。
```

除非该血字已经在线索表或场景证据中存在。

---

### 14.3.3 新闻包装

AI可以把城市事件包装成新闻。

系统输入：

```json
{
  "event_type": "power_outage",
  "district": "西港区",
  "affected_location": "白塔公寓",
  "official_reason": "线路老化",
  "hidden_reason": "远程调用旧电梯系统"
}
```

AI输出：

```text
白港晚报：西港区白塔公寓凌晨短暂停电，物业称原因是线路老化，未造成明显影响。
```

AI不能直接暴露隐藏原因。

---

### 14.3.4 图片提示词

AI可以生成场景图、证据图、监控图提示词。

必须遵守统一风格：

```text
第一视角
真实摄影
都市悬疑
竖屏
电影感
现代白港市
```

图片提示词不能包含真实人物、真实组织、真实地址、真实学校。

---

### 14.3.5 语音脚本

AI可以生成电话录音、语音留言、匿名举报文本。

必须包含：

- 说话人
- 情绪
- 环境音
- 可听见线索
- 转写文本

AI不能在语音中泄露系统未解锁真相。

---

## 14.4 AI禁止生成内容

AI禁止：

- 修改真相树根节点
- 新增关键真相
- 删除关键线索
- 改变NPC真实身份
- 改变NPC秘密
- 随意增加组织势力
- 随意增加真实地名
- 让NPC无理由说谎
- 让线索前后矛盾
- 生成无法校验的异常
- 直接告诉玩家最终答案
- 生成长篇世界观说明
- 生成复杂TRPG规则
- 生成强制多人玩法

---

## 14.5 AI输入结构

调用AI生成表现文本时，必须提供结构化输入。

示例：

```json
{
  "task": "generate_scene_dialogue",
  "case_id": "CASE_001",
  "scene_id": "SCENE_003",
  "scene_type": "chat",
  "case_level": "E",
  "known_truth": [
    "240号单位三年前已经空置",
    "凌晨2:13有人发送匿名私信"
  ],
  "hidden_truth": [
    "夜班保安删除过一段录像"
  ],
  "allowed_clues_to_reveal": ["CLUE_CHAT_001"],
  "forbidden_clues": ["CLUE_DELETED_VIDEO"],
  "npc_context": {
    "npc_id": "NPC_001",
    "name": "林知远",
    "role": "夜班保安",
    "emotion": "紧张",
    "lie_probability": 35,
    "can_lie_about": ["是否见过玩家"],
    "cannot_lie_about": ["是否在白塔公寓工作"]
  },
  "style_rules": {
    "length": "short",
    "tone": "urban_suspense",
    "perspective": "first_person_ui",
    "avoid_exposition": true
  }
}
```

---

## 14.6 AI输出结构

AI必须输出可解析JSON。

示例：

```json
{
  "scene_id": "SCENE_003",
  "generated_text": {
    "title": "夜班保安的回复",
    "messages": [
      {
        "speaker": "林知远",
        "text": "你别乱查。白塔公寓很多记录不是给住户看的。",
        "emotion": "压低声音"
      },
      {
        "speaker": "玩家",
        "text": "240号房到底有没有人住过？",
        "emotion": "追问"
      },
      {
        "speaker": "林知远",
        "text": "三年前就空了。至少系统里是这样写的。",
        "emotion": "迟疑"
      }
    ]
  },
  "revealed_clues": ["CLUE_CHAT_001"],
  "new_claims": [
    {
      "claim": "240号房三年前就空了",
      "source": "NPC_001",
      "matches_allowed_truth": true
    }
  ],
  "safety_flags": []
}
```

禁止AI输出自由散文后再由系统猜测含义。

---

## 14.7 AI生成步骤

标准流程：

```text
1. 读取系统输入
2. 确认允许暴露的信息
3. 确认禁止暴露的信息
4. 生成短文本
5. 标记已暴露线索
6. 标记新增说法
7. 自检是否越权
8. 输出JSON
```

---

## 14.8 信息暴露规则

每个场景只能暴露当前允许的信息。

信息分级：

```text
L0 玩家已知
L1 当前场景可获得
L2 当前案件后续可获得
L3 隐藏真相
L4 暗线主线信息
L5 S级事件核心信息
```

AI只能输出L0和L1。

AI不能主动输出L2以上信息。

除非系统明确写入：

```json
{
  "allowed_disclosure_level": "L3"
}
```

---

## 14.9 NPC说谎规则

NPC不能随便说谎。

NPC说谎必须满足至少一个条件：

- 有秘密
- 有恐惧
- 有利益损失
- 被组织威胁
- 想保护某人
- 想误导玩家
- 正在执行暗线任务

说谎必须记录：

```json
{
  "lie_id": "LIE_001",
  "npc_id": "NPC_001",
  "scene_id": "SCENE_003",
  "lie_text": "我没看过那段录像",
  "truth": "他看过并删除了录像",
  "reason": "害怕失去工作并暴露暗线接触",
  "can_be_exposed_by": ["CLUE_BACKUP_VIDEO"]
}
```

没有可揭穿方式的谎言禁止生成。

---

## 14.10 AI对白风格规则

《暗线》对白必须：

- 短
- 直白
- 有信息量
- 有悬念
- 有情绪
- 不解释世界观
- 不堆设定

推荐：

```text
“别去24楼。”

“为什么？”

“三年前那里就封了。但昨晚，电梯停过一次。”
```

不推荐：

```text
白港市是一座现代化、国际化、高度数字化的海港城市，在这座城市背后隐藏着大量不为人知的秘密……
```

玩家打开游戏后必须快速进入事件。

---

## 14.11 AI场景长度规则

### 聊天场景

```text
单条消息：5~35字
单场消息：3~8条
玩家选项：2~4个
```

### 电话场景

```text
通话文本：60~180字
必须包含至少一个可分析线索
```

### 新闻场景

```text
标题：12~24字
摘要：40~100字
评论：2~5条
```

### 档案摘要

```text
结案摘要：100~250字
调查员备注：30~80字
```

## 14.12 图片提示词规则

图片提示词必须包含：

```json
{
  "image_type": "evidence_photo",
  "perspective": "first_person",
  "style": "realistic urban suspense photography",
  "orientation": "vertical",
  "location": "白塔公寓24楼走廊",
  "visible_elements": ["昏暗走廊", "旧门牌", "电梯指示灯"],
  "hidden_detail": "门牌反光中有模糊黑影",
  "mood": "冷色调，压抑，电影感",
  "forbidden": ["真实地名", "真实人物", "血腥细节", "过度恐怖"]
}
```

禁止图片提示词擅自添加关键线索。

---

## 14.13 语音提示词规则

语音证据必须包含：

```json
{
  "audio_type": "anonymous_tip",
  "speaker": "未知男声",
  "emotion": "压低声音，紧张",
  "background_sounds": ["雨声", "电梯提示音", "远处广播"],
  "transcript": "别查物业系统，查旧备份。",
  "clue_tags": ["电梯提示音", "旧备份"],
  "forbidden_disclosures": ["暗线真实结构", "最终真相"]
}
```

## 14.14 AI自检清单

每次AI生成后必须自检：

```text
1. 是否暴露了禁止信息？
2. 是否新增了系统没有的线索？
3. 是否改变了NPC身份？
4. 是否改变了案件真相？
5. 是否生成了无法验证的异常？
6. 是否文字过长？
7. 是否缺少玩家选择？
8. 是否有真实人物、真实组织、真实地址？
9. 是否出现复杂规则说明？
10. 是否仍然符合都市悬疑短剧节奏？
```

如果任一项失败，AI输出必须被丢弃并重新生成。

---

## 14.15 AI输出校验器

系统应对AI输出进行机器校验。

校验字段：

```json
{
  "validation": {
    "truth_changed": false,
    "new_unregistered_clues": false,
    "forbidden_info_leaked": false,
    "npc_identity_changed": false,
    "scene_too_long": false,
    "missing_player_options": false,
    "invalid_real_world_reference": false,
    "pass": true
  }
}
```

只有 `pass: true` 的内容可以进入游戏。

## 14.16 推荐Prompt结构

系统调用AI时建议使用以下结构：

```text
你是《暗线》的剧情表现生成器。

你只能根据输入生成表现文本。
你不能修改真相。
你不能新增关键线索。
你不能暴露禁止信息。
你必须输出JSON。

【案件上下文】
...

【当前场景】
...

【允许暴露】
...

【禁止暴露】
...

【NPC状态】
...

【输出格式】
...
```

## 14.17 错误输出示例

错误：

```text
林知远其实是暗线成员，他知道这一切都是观察局安排的。
```

问题：

```text
提前暴露真实身份和组织信息。
```

正确：

```text
林知远看了一眼监控室的门，声音压低：“有些记录，不该从系统里看。”
```

---

## 14.18 核心原则总结

AI不是主持人。

AI不是真相设计者。

AI不是规则裁判。

AI只是表现层。

《暗线》的生成架构必须始终保持：

```text
系统决定发生了什么。
AI决定它看起来像什么。
玩家决定相信什么。
```

这是无限案件能长期稳定运行的基础。
