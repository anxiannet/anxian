# 01. Case Engine Overview

## 1. 目标

Case Engine 是《暗线》的核心内容引擎。

负责：

- 生成案件
- 驱动调查
- 管理线索
- 控制节奏
- 触发反转
- 归档结果
- 推进玩家成长
- 扩展白港市世界

它不是传统 TRPG 系统。

它是：

> 都市调查 × AI剧情 × 档案推理

的运行框架。

---

## 2. 核心设计理念

《暗线》的目标不是：

❌ 复杂规则

❌ 长篇设定

❌ 数值刷怪

❌ 多人依赖

而是：

✅ 一个人也能调查

✅ 5分钟卷入谜团

✅ 10分钟完成一次案件

✅ 无限生成案件

✅ 长期探索白港市

---

## 3. 玩家循环

收到异常信息 → 进入案件 → 调查场景 → 获得线索 → 发现矛盾 → 推理判断 → 获得真相 → 档案归档 → 触发新事件

所有系统都必须服务这个循环。

## 5. 案件生成流程

事件触发 → 选择案件模板 → 生成NPC → 生成真相树 → 生成线索 → 生成误导 → 生成反转 → 生成场景 → 生成图片 → 生成语音 → 输出案件

---

## 6. 真相优先原则

案件生成顺序：

真相 → 反转 → 误导 → 线索 → 场景

禁止先写场景再拼真相。

---

## 7. 白港市原则

所有案件发生于虚构城市：白港市（White Harbor）。

白港市是一座现代化海港城市。

表面正常。

暗地里存在大量异常事件、都市传说与暗线网络。

---

## 8. AI生成原则

AI负责：

- 案件导演
- NPC扮演
- 场景生成
- 线索控制
- 真相隐藏

所有结论必须：

- 可追溯
- 可验证
- 可归档

---

## 9. 长期内容循环

案件 → 档案 → 都市传说 → 暗线节点 → 隐藏案件 → 新都市传说 → 更深层暗线 → 案件
